## 2026-06-01 実施した基盤修正

| 領域 | 実施内容 | 目的 |
|------|----------|------|
| GA4 | `scripts/ga4-fetch.py` を期間指定対応に変更 | 月次・累計・直近30日を都度取得できるようにする |
| 技術SEO | `sitemap.xml` を111 URLへ再生成、`about/privacy` などnoindexページを除外 | Search Consoleの「送信URLがnoindex」「URL不一致」ノイズを減らす |
| 技術SEO | dog/cat詳細ページのURLを `/dog-names-toy-poodle` 形式に統一 | sitemap・canonical・dist実体の不一致を解消 |
| 技術SEO | canonical / og:url の `.html` 付きURLをクリーンURLへ統一 | Googleに正規URLを明確に伝える |
| 技術SEO | `guide/dog`, `guide/cat`, `guide/first` のビルド入力を修正 | sitemapに載るURLが実際に配信される状態にする |
| 計測 | GA4がCSPでブロックされないよう `vercel.json` を修正 | 本番でPV・クリック計測を取りこぼさない |
| 収益導線 | `/starter-set` をnoindex解除し、楽天カテゴリ導線ページへ変更 | Adsense以外のキャッシュポイントを持つページにする |
| 収益導線 | 高意図記事5本に楽天カテゴリ導線を追加 | 購買意図がある記事から楽天遷移を作る |
| 計測 | 楽天リンククリックを `click_rakuten_affiliate` でGA4送信 | どの記事・カテゴリが収益導線として強いか判断する |
| 収益導線 | 楽天導線を発行済みURL優先・検索URLフォールバック方式に変更 | 楽天アフィリエイト管理画面で発行したURLを `url` または `href` に入れるだけで差し替えられるようにする |
| アプリ導線 | App Store未公開のプレースホルダリンクを公開HTMLとアプリ設定から削除 | 壊れた外部リンクを出さず、公開まではWeb版診断へ送る |

## 現時点のキャッシュポイント整理

| 導線 | 現状 | 次の改善 |
|------|------|----------|
| 楽天アフィリエイト | `starter-set` と高意図記事に楽天市場カテゴリ導線を配置。`src/affiliate-links.js` は発行済みURLを優先 | 楽天アフィリエイト管理画面で発行したURLを各itemの `url` または `href` に追加する |
| ペット名前アプリ | HP上にアプリ紹介バナーあり。App Store ID未確定のため、今はWeb版診断へ送る | App Store公開後に実URLへ差し替え、診断結果直後のCTAを強化 |
| ペット生活記録アプリ | 2つ目のアプリバナーは非表示。App Store未公開のため、お迎え準備ガイドへ退避 | 公開URL確定後に表示し、診断結果名の保存導線とつなぐ |
| Adsense | 使わない方針 | 当面は入れない。SEO記事から楽天・アプリ導線へ送る |

