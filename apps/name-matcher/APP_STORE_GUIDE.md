## 1. ペルソナ分析

### ペルソナ A: 初めてペットを迎える20-30代女性「まい」
- **属性**: 東京都内在住、一人暮らし、IT企業勤務
- **行動**: 保護犬/保護猫の里親になる。Instagramで #ペットの名前 を検索
- **課題**: "かわいい名前がありすぎて決められない" "家族や友人と意見が割れる"
- **きっかけ**: SNSで「スワイプでペットの名前診断」を見てインストール
- **使い方**: 種類と雰囲気だけ指定 → ひたすらスワイプ → 上位3候補を家族LINEで共有
- **期待**: 自分の好みに合った名前を提案してくれること、共有が楽なこと

### ペルソナ B: 多頭飼いの40代女性「ゆかり」
- **属性**: 神奈川県在住、一戸建て、パート勤務
- **行動**: 犬3匹・猫1匹を飼育。すでに何度も命名経験あり
- **課題**: "これまでと同じ系統の名前ばかり選んでしまう" "新しいテイストに挑戦したい"
- **きっかけ**: 動物病院の待合室でポスター/チラシを見る
- **使い方**: あえて雰囲気を変えてスワイプ → 今まで選ばなかった名前を発見
- **期待**: マンネリ打破、新鮮な名前との出会い

### ペルソナ C: ペットの名前で悩む家族（母「かな」）
- **属性**: 埼玉県在住、小学生の子供あり
- **行動**: 家族会議で名前を決めたいが、子供と意見が合わない
- **課題**: "子供が気に入る名前と自分が良いと思う名前が違う"
- **きっかけ**: Webで「ペット 名前 診断」と検索 → sippomi.com を発見 → アプリへ
- **使い方**: 家族で1枚ずつスワイプ → 結果画面を見ながら家族会議 → 保存候補から最終決定
- **期待**: 家族で楽しく使えるUX、視覚的に分かりやすい結果

## 4. ビルド・提出手順

### 4.1 EAS Build
```bash
# iOSビルド
cd apps/name-matcher
npx eas build --platform ios --profile production

# またはローカルビルド
npx expo prebuild --platform ios
npx expo run:ios --configuration Release
```

### 4.2 TestFlight配信
```bash
npx eas submit --platform ios
```

### 4.3 App Store Connect 設定
1. アプリ情報 → プライバシーポリシーURL: `https://sippomi.com/privacy`
2. アプリのバージョン → 審査用メモに以下を記載:
   - アカウント不要で全機能が利用可能
   - ネットワーク非依存のオフライン対応アプリ
3. 輸出コンプライアンス → ITSAppUsesNonExemptEncryption = false
